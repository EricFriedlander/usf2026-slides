🎲 Graduate School: Large Networks

Probability & asymptotic methods for massive systems

Modeling large systems

  • Suppose we have a system of \(n\) “particles”
  • Particles can be anything (e.g. queues, individuals, molecules, etc…)
  • Each partical is evolving according to some random process
  • Modeling each of these particles directly \(\Rightarrow\) \(n\)-dimensional process (infeasible for large \(n\))
  • Solution: Assume \(n\) is infinite
  • Umm… wut?

Asymptotic Approximations

Fluid: continuous limit of discrete molecules

Waves: aggregate behavior, not individual particles

Asymptotics Approximations

  • A few general types of asymptotic approximations (you’ve heard of two)
    1. Law’s of large numbers (limit is deterministic)
    2. Central limit theorem (limit is stochastic)
    3. Large deviation theorem (how fast does LLN converge)

Load Balancing

Routing tasks in a large server network

Load Balancing

Key question: What’s the best way to route tasks in a large server network?

Random Routing

Assign each task to a random server — simple, but inefficient

Power of Two Choices

Sample 2 servers, pick the shorter queue

Power of Two Choices

Join-the-Shortest-Queue

Always join the shortest queue — optimal, but expensive to implement

How do we evaluate this for lots of servers?

  • LLN \(\Rightarrow\) ODE on distribution of queue lengths
    • What does the steady-state distribution of this ODE look like?
      • JSQ: all queues are of length 0 or 1
      • Random routing: queue lengths decay exponentially
      • Power-of-two: queue lengths decay double-exponentially

Steady-State Distributions

What about the impact of randomness?

  • CLT \(\Rightarrow\) SDE for distribution of queue lengths
  • This my work (specifically for power-of-two) \(\Rightarrow\) about 40-pages of proofs… none of which I’ll present here
  • SDE allows for more efficient simulation

What about this Large-Deviation Theorem?

I proved a LDT for JSQ.

Theorem. \[\begin{align} \lim_{n\to\infty}\frac{1}{n}\log\mathbb{P}(X^n\in G) &= \inf_{x\in G}I(x) \end{align}\]

Useful in estimating probabilities of rare-events.

Sequencing Data

The modeling framework

Goal: turn data into answers about history

\[\text{Assumptions} + \text{Data} \xrightarrow{\text{model}} \text{Likelihood (Probability)}\]

  • Assumptions: population size history, mutation rate, selection strength
  • Data: sequencing data from real individuals
  • Output: probability of seeing that data under our assumptions

Vary the assumptions → find the ones that make the data most probable

Let’s build a model!

Wright-Fisher Model: Setup

  • Focus on one position in the genome
  • Two versions (“alleles”) at that site:
    • Mario and Luigi
  • Population size N (fixed each generation)
  • Time moves in discrete generations
  • Each offspring randomly picks a parent from the previous generation
  • Track \(p_k\) = proportion of Luigis in generation \(k\)

  • N = 1,000 individuals?
  • Each generation: sample 1,000 parents (with replacement)

Baseline Correction

  • Instrument reads a “floor” even with no signal
  • Asymmetric least squares fits and removes this floor
  • Think: subtract a wavy background from a photo

\[ \min_{z} \; \sum_{i=1}^{n} w_i \, (y_i - z_i)^2 \;+\; \lambda \sum_{i=2}^{n-1} (z_{i-1} - 2 z_i + z_{i+1})^2 \]

where:

  • \(y_i\) : observed signal at point \(i\)
  • \(z_i\) : estimated baseline at point \(i\)
  • \(w_i\) : asymmetric weights, updated iteratively:

\[ w_i = \begin{cases} p, & \text{if } y_i > z_i \\[2mm] 1 - p, & \text{if } y_i \le z_i \end{cases} \]

  • \(\lambda\) : smoothing parameter controlling baseline smoothness
  • \(p\) : asymmetry parameter (small values keep the baseline below peaks)

📊 Psychometrics

Measuring the unmeasurable

A book recommendation

One of the best textbooks I’ve ever read

Basic questions in Psychometrics

  • How do we determine if a theory is real?
  • How do we decide if someone has learned something?
  • How do we decide if someone feels something?
  • How do we determine if it is possible to feel something?
  • How do we measure things?

Construct

Start with a construct

Developing an assessment

  • Design an exam or survey to assess your construct
    • Each question is called an item in psych-talk
  • Is your assessment good?
    • Reliability: is the measurement consistent?
    • Validity: does it measure what it’s supposed to?

Reliability

If I measure individuals with the same trait, do they score the same?

  • Test-retest reliability: if you measure someone twice… do they get the same score
  • Internal consistency: does each question positively correlate with total score?
    • Chronbach’s Alpha: \(\alpha = \frac{k}{k - 1} \left( \frac{\sum_{i\neq j} \sigma^2_{y_iy_j}}{\sigma^2_x} \right)\)
  • Inter-Rater Reliability: if two people are scoring an assessment

Validity

Many types of validity (Coopers lists 12):

  • Content Validity: are the items a representative sample of the domain they measure
  • Criterion-Related Validity: how well do test scores correlate with an external, “real-world” outcome (the criterion)
  • Construct Validity: the extent to which a test measures the theoretical construct it was designed to measure

Statistical Techniques

  • Factor analysis & PCA
    • Exploratory Factor Analysis (EFA)
    • Confirmatory Facotr Analysis (CFA)
  • Item-Response Theory (IRT)

Factor Analysis & PCA

  • PCA: take correlation matrix \(\to\) compute SVD
  • Factor Analysis:
    • Assume some generative model: \(\mathbf{x} = \mathbf{\Lambda} \mathbf{f} + \boldsymbol{\epsilon}\)
      • \(\mathbf{x}:\) data
      • \(\mathbf{\Lambda}:\) loadings (constant across all individuals)
      • \(\mathbf{f}:\) factor scores
      • \(\boldsymbol{\epsilon}:\) error
    • Partition variance: \(\mathbf{\Sigma} = \mathbf{\Lambda} \mathbf{\Lambda}^T + \mathbf{\Psi}\)
    • Note \(\mathbf{\Lambda}\) not unique so we must “rotate”

What Does Correlation Mean?

  • When X goes up, Y tends to go ___
  • What does it mean for a variable to be quantitative?
  • Are Likert scores quantitative?
  • Suppose \(X\) is a normal random variable and \(Y\) is a binary variable (0/1) can they ever have a correlation of 1?

The Latent Variable Assumption

What polychoric/tetrachoric actually assume

  • Observed categories = discretization of a hidden continuous variable
  • Latent variable assumed to be normally distributed

The Latent Variable Assumption

Example: “Rate your pain 1–5”

  • Your actual pain exists on a continuous scale
  • You cross a threshold → you report the next category up

Hidden number line:

——|——|——|——|——

1 2 3 4 5

Thresholds (\(\tau\)) carve it into bins

  • We only observe which bin
  • Latent value is inferred
  • Polychoric recovers the latent correlation

Item Response Theory (IRT)

Item Characteristic Curve (ICC):

\[P(\text{correct} \mid \theta) = \frac{1}{1 + e^{-\alpha(\theta - \beta)}}\]

  • Difficulty (\(\beta\)): where on ability scale does P(correct) = 0.5?
  • Discrimination (\(\alpha\)): how steeply does the curve rise?

IRT: Why It Matters

Advantages over classical test theory

  • Item parameters don’t depend on the sample
  • Person ability doesn’t depend on which items used
  • Used by: GRE, SAT, LSAT, AP exams

A student taking a harder test version gets a fair ability estimate

The Big Caveat

All of this assumes (at least latent) continuity

  • What if variables are truly categorical?
    • Not discretized — genuinely unordered
    • Polychoric/IRT assumptions break down
  • Correlations and factor loadings become misleading
    • Artificially inflated or deflated
    • Factors may reflect response style, not constructs

💡 Random Thoughts

Things that don’t fit elsewhere (but should)

What is the purpose of a mean?

  • What is a mean?
  • Why is the mean so important?
  • When should we use the mean descriptively? (Don’t say anything about outliers)
  • My answer: when we’re going to repeat some process over and over again…

Easy Example

Every month 100,000’s of customers come into my store. Suppose the money that customer \(i\) spends is represented as \(x_i\)… How should I compute the mean of \(x_i\)?

Arithmetic mean: \(\bar{x} = \frac{\sum x_i}{n}\)

Harder Example

I have a 403B (retirement account). Suppose my the interest rates I’ll receive over the next 20 years are represented as \(x_i\)… How should I compute the mean of \(x_i\)?

Note: \(\text{Balance}\times (1 + \bar{x})^{20} \neq \text{Balance}\times (1 + x_1)\times\cdots\times(1 + x_{20})\)

Geometric Mean

\[M = \left(\prod_{i=1}^{n} x_i\right)^{1/n}\]

  • Used when things compound or grow or shrink
  • Finance (1 + interest rates)
  • Population genetics (what was the average population size?)

Hardest Example

Suppose I drive from SF to LA at 80 MPH and then drive back at 60 MPH… What is my average speed?

  • It depends… am I weighting based on distance or time?
  • Distance: arthmetic mean.
  • Time… harmonic mean

Harmonic Mean

\[M = \frac{n}{\sum_{i=1}^{n} \frac{1}{x_i}}\]

  • Let’s think about units

\[ \frac{n}{\sum_{i=1}^{n}\frac{1}{\text{(Miles per hour)}}}= \frac{n}{\sum_{i=1}^{n} \frac{\text{Hours}}{\text{Miles}}} \]

  • Use whenever you’re averaging rates but weighting by the denominator
  • F1-Score: Harnomin mean of precision and recall

Should You Scale Dummy Variables?

Setup: one-hot encoding

  • Convert categorical → binary dummy variables
  • “Color: Red/Blue/Green” → is_red, is_blue, is_green
  • Each column is 0 or 1

The question: after encoding, should you center and scale these binary columns?

Short answer: it depends on the algorithm

The KNN Problem

2 features:

  • age
  • dogs_or_cats

After standard scaling (x - mean) / sd:

  • age: stays roughly -3 to 3
  • dogs_or_cats: \(\frac{p}{\sqrt{p(1-p)}}\)

What if we ask about favorite animal?

  • Suppose instead that we ask “What is your favorite animal?”
  • We get 20 different responses…
  • What is the largest possible \(p\) for the least frequent category?
  • Moral of the story… probably don’t scale for KNN (can scale to zero-one)

The LASSO Problem

LASSO penalty: \(\lambda\times |\beta_j|\) for each coefficient

  • If we don’t scale: \(\beta_j\) will be in units of response variable (maybe really big)
  • If we do scale… \(x_j\) will be come very largeand \(\beta_j\) will become very small (i.e. not get penalized)
  • So what do we do?
  • I honestly don’t know… but this idea applies to a variety of problems(e.g. feature importances, feature selection, normalization, etc.)

🌟 Star Formation

Machine learning for astrophysics

What Are Young Stellar Objects?

Young Stellar Objects (YSOs)

  • Protostars still in the process of forming
  • Embedded in dense clouds of gas and dust
  • Visible light is blocked — only detectable in infrared
  • Studying YSOs reveals how stars like our Sun formed

The MIRION “Yellowballs” Catalog

Named for their distinctive yellow-green appearance in Spitzer Space Telescope infrared images, Yellowballs are a population of compact YSO candidates.

Can we predict their physical properties from their light alone?

The Data

43 Features from 4 observatories

  • 10 infrared flux measurements spanning 8–1100 μm
  • Per-band photometric error and quality columns
  • Sources:
    • Spitzer Space Telescope
    • Herschel Space Observatory
    • ALMA / SCUBA submillimeter observations

The Pipeline: Bayesian Model Search

For each of the 8 targets, an independent pipeline was run:

  • BayesSearchCV (150 iterations): efficient hyperparameter search across a large space — smarter than grid search
  • 6 model types searched simultaneously: Random Forest, Gradient Boosting, XGBoost, LightGBM, ElasticNet, MLP
  • Preprocessing pipeline: imputation → feature scaling → model
  • Best single model selected and saved per target
  • Stacking ensemble: all 6 models combined with a Ridge meta-learner for final predictions

Running at Scale: Borah HPC

Each target = 150 BayesSearch iterations × 5-fold CV = 750 model fits

Submitted as a SLURM array job on Boise State’s Borah HPC cluster — one job per target.

Resource Per Job
CPUs 48 cores
RAM 128 GB
Wall time up to 24 hrs

The Power of Parallelism

All 8 targets ran simultaneously:

48 CPUs × 8 jobs = 384 cores working in parallel

  • What would take weeks on a laptop runs overnight on a cluster
  • Wall time: ~24 hrs total vs. ~192 hrs (8 days) sequentially
  • HPC makes systematic hyperparameter search practical

Which Wavelengths Drive Temperature Predictions?

SHAP values show how each feature pushes the prediction. The 160 μm and 350 μm far-infrared bands dominate — consistent with Wien’s law (cooler objects emit more at longer wavelengths).

Not All Properties Are Equal

Diameter: R² = 0.093

Diameter depends on distance assumptions — distance errors propagate directly into this target

Some properties simply aren’t encoded in the photometry alone

What We Learned

  • Bayesian hyperparameter search efficiently explores complex model spaces
  • XGBoost consistently outperforms simpler models on this dataset
  • Some physical properties (temperature, luminosity ratio) are reliably predictable from infrared photometry
  • Others (diameter) require additional data or better distance estimates
  • HPC enables systematic ML research that would be impractical on a laptop
  • Stacking ensembles offer modest gains over single best models

Infrared light is a remarkably rich signal — but not every stellar secret is written in photons
