Modeling large systems

  • Suppose we have a system of \(n\) “particles”
  • Particles can be anything (e.g. queues, individuals, molecules, etc…)
  • Each partical is evolving according to some random process
  • Modeling each of these particles directly \(\Rightarrow\) \(n\)-dimensional process (infeasible for large \(n\))
  • Solution: Assume \(n\) is infinite

File Caching: gLRU

Storing files efficiently in a large network

The File Caching Problem

Key question: What’s the best way to store files in a large network?

  • Standard LRU (Least-Recently-Used): evict oldest file
  • Problem: files have different sizes
  • Large files waste cache space
  • Small files get unfairly evicted

gLRU: Chunk-Based Caching

Standard LRU

  • Files treated as atomic units
  • One big file crowds out many small ones

gLRU

  • Subdivide files into equal-sized chunks
  • Add chunks of requested file
  • Remove chunks of least-recently-used file

→ Cache space used far more uniformly

gLRU: Key Results

  • ~32% reduction in video streaming stall duration vs. standard LRU
  • Validated on real-world data (MSR Cambridge Traces)
  • Published in ACM TOMPECS (2019)

Applications: Content delivery networks — Netflix, YouTube, Cloudflare
